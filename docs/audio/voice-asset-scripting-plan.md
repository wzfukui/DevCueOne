# DevCue One 语音资产脚本计划

## 文档目的

记录后续要补的语音脚本工具约定，但当前阶段先不实现脚本。

目标是让后续的语音样本生成与转录调用，有统一的本地执行方式。

---

## 约定方向

后续可以增加两类本地脚本：

1. 语音合成脚本
2. 语音转录脚本

形式建议：

- `scripts/audio/generate-tts.sh`
- `scripts/audio/transcribe-audio.sh`

## 环境变量约定

密钥不写入脚本本体，不写入仓库文档内容，统一从本地 `.env` 注入。

示例变量名建议：

- `OPENAI_API_KEY`
- `OPENAI_TTS_MODEL`
- `OPENAI_TTS_VOICE`
- `OPENAI_STT_MODEL`

## 预期用途

### 1. 生成测试语音样本

- 根据固定文本清单批量生成中文 / 英文音频
- 用于自动化测试与人工验收

### 2. 生成短确认语音资产

- 根据确认语音台词清单批量生成本地音频资产
- 存入固定目录供运行时直接播放

### 3. 语音转录验证

- 对固定音频样本执行转录
- 校验 STT provider 与工作语言配置效果

## 设计要求

- 支持从 `.env` 加载配置
- 支持指定输入文本文件或目录
- 支持指定输出目录
- 支持 dry-run
- 支持错误码返回
- 支持在日志里输出：
  - provider
  - model
  - language
  - input / output path

- 脚本实现放在后续单独步骤
