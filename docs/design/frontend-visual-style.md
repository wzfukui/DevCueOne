# DevCue One 前端视觉风格说明

## 1. 一句话定义

当前界面的风格可以概括为：

暖中性色纸面质感 + 轻玻璃化面板 + 编辑排版感控制台。

它不是典型的深色科技面板，也不是强烈霓虹式 AI 风格，而是一种更克制、更耐看、偏专业工作台的桌面界面语言。

## 2. 风格关键词

- Editorial Control Panel
- Warm Neutral Dashboard
- Soft Glass Utility UI
- Calm Ops Console
- 中文语境下的“纸面感桌面控制台”

如果要对外描述，可以直接说：

这是一套偏编辑排版感的桌面控制台风格，基底是暖纸色和低饱和中性色，配合轻微毛玻璃、圆角面板、蓝灰主按钮，以及琥珀色信息标签，整体感觉克制、专业、耐看，不走传统 AI 产品常见的深色霓虹路线。

## 3. 视觉气质

- 主观感受：安静、理性、可信、可持续使用
- 设计取向：弱炫技，强信息秩序
- 界面角色：长期驻留的桌面工作台，而不是一次性营销页
- 用户心理：让人愿意一直开着，不会因为颜色太刺激而疲劳

## 4. 色彩体系

### 基础方向

- 背景不是纯白，而是带暖黄纸感的浅底色
- 中和色里混入少量蓝灰，避免页面发黄或发闷
- 强调色不是高饱和霓虹，而是克制的蓝、琥珀、绿色、红色状态色
- 大面使用低对比柔和层次，小面积交互再给更明确的强调

### 当前界面的核心色彩角色

| 角色 | 用途 | 代表色 |
| --- | --- | --- |
| Paper Base | 页面主背景 | `oklch(0.98 0.015 95)` 到 `oklch(0.94 0.018 95)` |
| Warm Glow | 左上暖色氛围光 | `oklch(0.92 0.07 85 / 0.85)` |
| Cool Glow | 右上冷色平衡光 | `oklch(0.92 0.06 205 / 0.75)` |
| Surface | 主面板背景 | `oklch(0.985 0.01 95 / 0.84)` |
| Border | 面板边界 | `oklch(0.81 0.02 85 / 0.72)` |
| Main Text | 主文字 | `oklch(0.22 0.035 255)` 到 `oklch(0.26 0.03 250)` |
| Accent Label | 小标题/眉标 | `oklch(0.5 0.09 55)` |
| Primary Action | 主按钮渐变 | `oklch(0.32 0.05 250)` 到 `oklch(0.41 0.08 210)` |
| Success | 正常监听状态 | `oklch(0.74 0.13 145)` |
| Warning | 转写/处理中 | `oklch(0.7 0.16 75)` |
| Speaking | 播报状态 | `oklch(0.68 0.16 230)` |
| Error | 错误状态 | `oklch(0.62 0.2 25)` |

### 用色原则

- 不用纯黑纯白，所有颜色都稍微带一点色相
- 主体颜色靠中性色和纸面色撑住，强调色只用于关键状态
- 冷暖同时存在，但暖色为主、冷色为辅
- 按钮和动态状态用蓝绿橙红区分，不把所有“高亮”都做成同一种色

## 5. 字体系统

### 当前搭配

- 正文：`IBM Plex Sans`
- 标题：`Fraunces`
- 中文兜底：`PingFang SC` / `Noto Sans SC` / `STSong`
- 等宽信息：`IBM Plex Mono`

### 字体角色分工

- `Fraunces` 负责标题气质，带一点编辑感和高级感
- `IBM Plex Sans` 负责正文和控件，可读性稳定，适合工具型界面
- `IBM Plex Mono` 只放在时间、路径、日志、细节信息上，强化“系统反馈”感

### 字体风格原则

- 标题要有明确人格，不要回退成普通系统无衬线
- 正文字体要稳，不要过度装饰
- 等宽字体只在技术信息位出现，避免全界面工程化过重

## 6. 布局语言

### 总体结构

- 左侧是主任务区：补充文本、会话记录、诊断信息
- 右侧是辅助状态区：HUD、摘要信息、最近捕获、设置
- 信息分层明确：高频操作靠前，低频配置下沉

### 布局关键词

- 主副分栏
- 卡片化但不过度堆卡
- 粘性关键控制
- 长列表与即时状态分离
- 首屏只保留高频内容

### 具体规则

- 主内容区更宽，右栏更窄，突出“任务主线在左边”
- 关键操作卡允许 `sticky`，但不要让所有卡片都粘住
- 右栏信息要按频率拆层，而不是全部塞进一个超级 HUD
- 最近内容和系统状态要分组，不要混在一起

## 7. 组件外观规则

### 面板

- 大圆角，当前主面板约为 `1.6rem` 到 `2rem`
- 半透明浅色背景
- 轻边框
- 轻阴影
- 带一点 `backdrop-filter: blur(...)` 的玻璃感，但克制使用

### 按钮

- 主按钮：蓝灰渐变，适合作为“开始/停止/保存”等主动作
- 次按钮：浅纸色底，轻边框，避免抢主按钮焦点
- 整体使用胶囊圆角，降低工具界面的生硬感

### 标签与说明文字

- 小标题统一大写、加大字距，偏琥珀色
- 说明文字统一蓝灰低对比，不抢主信息
- 时间、日志、路径等细节内容用等宽字体增强结构感

## 8. 动效与交互反馈

### 当前特点

- 按钮 hover 只做轻微上浮，不做夸张弹跳
- 声音/忙碌状态用细长进度条流动反馈
- 麦克风电平用渐变条而不是仪表盘式重装饰

### 动效原则

- 动效服务于状态理解，不服务于炫耀
- 反馈要短、轻、明确
- 不做重玻璃、重发光、重粒子

## 9. 这套风格适合什么场景

- 桌面常驻助手
- 工作台型 AI 产品
- 语音代理 / 编排台 / 控制台
- 偏专业而非娱乐化的内部工具
- 需要长时间打开使用的生产力界面

## 10. 不适合什么方向

- 强营销导向的 Landing Page
- 赛博朋克/霓虹/全暗色 AI 宣传页
- 年轻化强娱乐产品
- 需要高度游戏化、情绪化表达的界面

## 11. 复用时要保持的核心规则

- 背景始终保持“暖纸底 + 冷暖双氛围光”
- 标题继续使用有个性的衬线或类衬线字体
- 面板始终浅底、轻边框、轻阴影、轻模糊
- 主按钮继续走蓝灰系，不改成高饱和霓虹
- 状态色分工明确：正常、处理中、播报、错误必须能一眼区分
- 右栏永远按信息频率分层，不要回退成一个大杂烩面板

## 12. 给别人沟通时可直接复用的描述模板

可以直接把下面这段发给设计师或前端：

请沿用当前这套“暖中性色编辑型控制台”视觉风格：整体是浅色纸面质感背景，叠加轻微暖色和冷色氛围光；面板使用半透明浅底、细边框、柔和阴影和少量模糊；标题采用有性格的衬线体，正文用稳定无衬线，日志和时间用等宽字体。交互颜色以蓝灰为主，辅以琥珀标签和明确的状态色。整体目标不是炫技型 AI 界面，而是一个可以长期停留、安静专业、层级清楚的桌面工作台。

## 13. 如果以后要继续演进

优先保持这三点不变：

1. 暖纸色背景和冷暖混合氛围
2. 标题的编辑感字体组合
3. 信息层级先于视觉装饰

如果要扩展新页面，建议继续沿用同一套色相逻辑和组件气质，而不是换成另一种完全不同的科技风。
