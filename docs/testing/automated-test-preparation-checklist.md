## A. 阻塞项

这些是进入“可持续自动化测试”前最关键的准备。

### A1. 测试策略边界

我需要你明确 3 个策略：

- 自动化测试默认是否允许真实联网
- 自动化测试是否允许真实调用 OpenAI
- 自动化测试是否允许真实调用 Codex CLI

推荐默认值：

- 默认不走真实外网
- 默认不用真实 OpenAI
- 默认不用真实 Codex
- 以 stub / fake / fixture 为主

原因：

- 结果更稳定
- 速度更快
- 不会把额度和登录态绑死在 CI 上

### A2. 测试操作系统范围

我需要你确认目标测试平台：

- 只做 macOS
- macOS + Windows
- macOS + Windows + Linux

推荐默认值：

- 第一阶段至少要有 macOS

原因：

- 这个产品依赖麦克风与 Electron 权限链路
- macOS 是权限、签名、打包差异最明显的平台

### A3. 样例项目目录

我需要至少 2 到 3 个稳定的本地测试目录，用来模拟 profile 绑定。

建议准备：

- `/workspace/project-alpha`
- `/workspace/project-beta`
- `/workspace/project-gamma`

每个目录最好：

- 结构稳定
- 不会频繁改动
- 不含敏感信息

### A4. 测试音频样本

我需要一组固定的中文语音样本。

建议至少准备：

- 10 条正常语音指令
- 3 条空白 / 噪音 / 短促无效输入
- 3 条本地路由指令
  - 切会话
  - 暂停监听
  - 打开目录 / 浏览器
- 3 条容易混淆的句子

每条最好有：

- 音频文件
- 文本标注
- 预期归类
  - 普通任务
  - 本地控制
  - 应被忽略

建议优先策略：

- 第一批先用人工录制真实中文语音
- 第二批再补标准化合成语音样本，减少录音环境噪音差异

如果采用 API 生成语音样本：

- 只在本地执行
- 密钥不进入仓库
- 生成后的音频文件和标注可进入测试资产目录

### A5. 可替换的假后端策略

我需要你同意后续测试中引入以下替身：

- Fake Transcriber
- Fake Codex Runner
- Fake TTS

推荐默认行为：

- 假转写：输入固定音频，输出固定文本
- 假 Codex：按预设场景返回 done / failed / need_input / hanging
- 假 TTS：只记录请求，不真正播报

## B. 强烈建议准备

### B1. 测试专用配置与密钥策略

如果未来某些回归测试必须走真实服务，我需要：

- 一个专用测试 OpenAI Key
- 明确额度上限
- 允许使用的环境范围
  - 本地
  - CI
  - 仅人工回归

更推荐做法是：

- 真实服务只保留给人工烟雾测试
- 自动化仍以 fake/stub 为主

### B2. 任务替身样本

我需要几个可控的“假 Codex 任务场景”：

- 立即成功
- 需要 10 秒成功
- 永久挂起
- 返回格式错误
- 返回 need_input
- 被用户取消

这会极大提升任务调度和恢复逻辑的测试稳定性。

### B3. 权限测试机器或测试账号

如果要验证麦克风拒绝 / 已授权 / 被系统改回授权后的行为，最好准备：

- 一台可反复切换权限状态的测试机
- 或一套明确的权限切换步骤

尤其是 macOS：

- 首次授权
- 拒绝后再次请求
- 系统设置中改回允许
- 改权限后是否需要重启

### B4. 打包态测试目标

我需要知道你是否打算在 Phase 2 或之后尽快验证：

- `.dmg` / `.pkg` / Windows 安装包
- 自动更新
- 签名与 notarization

如果答案是“短期内要”，那测试设计需要提前把这部分纳入。

当前已确认：

- 短期不把打包态测试纳入主线验收阻塞项

## C. 需要你拍板的质量标准

这些不是资产，而是标准。

### C1. 哪些场景必须自动化

我建议至少自动化：

- 会话创建 / 切换 / 改名
- profile 创建 / 编辑 / 移除确认
- 新建会话必须选择项目
- 已创建会话不可切换项目
- 会话恢复
- 多会话任务归属
- 控制语义拆分
- 语音路由命中与回退

### C2. 哪些场景保留人工测试

我建议暂时保留人工：

- 真实麦克风输入体验
- 真实 TTS 播报自然度
- 真实 OpenAI 网络链路
- 打包后的首次权限弹窗体验

### C3. 失败时要保留哪些证据

我建议自动化失败时统一保留：

- 当前窗口截图
- 会话列表截图
- 诊断日志导出
- 数据库快照
- 对应 debug-audio 文件路径
